# Contributing to Opsimind

Thank you for your interest in contributing to Opsimind!

Opsimind aims to become an Operational Cognitive Platform that helps engineering teams understand, reason about, and optimize complex software systems.

We welcome contributions of all sizes, including documentation improvements, bug fixes, feature implementations, testing, and design discussions.

---

# Before You Start

Please:

- Read the project documentation.
- Search existing issues before opening a new one.
- Discuss significant architectural changes before implementation.

---

# Development Workflow

1. Fork the repository.
2. Create a feature branch.

```bash
git checkout -b feature/my-feature
```

3. Make your changes.
4. Add or update tests.
5. Ensure all tests pass.
6. Commit using meaningful commit messages.
7. Push your branch.
8. Open a Pull Request.

---

# Coding Standards

We value:

- Clean architecture
- Readable code
- Small focused pull requests
- Comprehensive tests
- Good documentation
- Backward compatibility whenever practical

---

# Documentation

Documentation is considered part of the product.

When adding new functionality, please update the relevant documentation.

---

# Pull Requests

A good pull request should include:

- Clear description
- Motivation
- Design decisions
- Testing performed
- Screenshots if applicable

---

# Reporting Issues

Please include:

- Environment
- Version
- Steps to reproduce
- Expected behavior
- Actual behavior
- Relevant logs

---

# Code of Conduct

Be respectful.

Constructive discussions and professional collaboration are expected from everyone participating in this project.

---

# Questions

If you have questions about architecture or roadmap, feel free to open a discussion before starting implementation.

Thank you for helping make Opsimind better!
